## Purpose

The Contribution Guide defines how contributors propose, prepare, review, document, and deliver changes to Guild CMS while preserving the engineering standards established by the Constitution, Vision & Mission, Engineering Principles, Architecture Standards, and Developer Handbook.

## Contribution Principles

Contributions should strengthen the implementation, the engineering record, and long-term maintainability of Guild CMS. Contributors should work through architecture discussion, Development Center alignment, implementation, security review, public documentation where applicable, and release packaging.

## Contribution Types

Guild CMS accepts code, documentation, bug reports, security findings, feature proposals, usability feedback, testing results, accessibility feedback, design suggestions, and operational notes.

## Review Expectations

Review is constructive, evidence-based, and tied to published project standards. A technically working change is incomplete when it leaves documentation, SQL, release notes, public site status, or Development Center status inconsistent.

## Security

Security-sensitive contributions require explicit review. Credentials, passwords, private keys, tokens, and sensitive operational details must never be included in release packages or public documentation.

## Release Packages

Release packages contain changed files only, required SQL, verification scripts, release notes, implementation guidance, security review, and package manifest.

## Stewardship

Every contributor is a steward of Guild CMS. Good contributions solve the immediate problem while making the project easier for future maintainers to understand and safely extend.
